# HaloClaw 电商改造方案

> 版本：1.0.0 | 日期：2026-04-17

## 一、项目概述

### 1.1 改造背景

HaloClaw 是基于字节跳动开源项目 UI-TARS-desktop（Apache 2.0 协议）改造的 Ozon 电商 AI 助手。UI-TARS-desktop 是一个 15.5k Stars 的 GUI Agent 桌面应用，原生支持浏览器自动化、MCP 协议集成和 AI 对话能力，非常适合用于电商自动化场景。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 品牌定制 | 从 UI-TARS 改造为 HaloClaw 电商助手 |
| 电商 Skills | 集成选品、货源、上架三大核心功能 |
| 用户认证 | 基于 Supabase 的用户注册/登录系统 |
| 计费集成 | 与 NewAPI 对接实现 AI 调用计费 |

## 三、Skills 集成方案

### 3.1 Skills 架构

```
haloclaw-tars/
├── skills/
│   ├── ozon-selection/      # Ozon选品Skill
│   │   └── SKILL.md
│   ├── source-1688/        # 1688货源匹配Skill
│   │   └── SKILL.md
│   └── ozon-listing/       # 商品上架Skill
│       └── SKILL.md
```

### 3.2 三大核心 Skills

#### Ozon-Selection（选品模块）

**功能**：蓝海产品智能筛选

**核心能力**：
- 竞争度分析（销量、评论、卖家数）
- 利润率评估（成本+运费+汇率）
- 趋势预测（历史数据分析）

**MCP 工具**：`ozon_pricing`（定价计算）

#### Source-1688（货源模块）

**功能**：1688 优质供应商匹配

**核心能力**：
- 关键词搜索/图搜货源
- 供应商筛选（诚信等级、成交量）
- 价格对比和拿样服务

**技术方案**：
- 1688 Open API（官方接口）
- Playwright GUI 爬虫（页面数据）
- Cookie 认证（需用户登录）

#### Ozon-Listing（上架模块）

**功能**：商品自动化上架

**核心能力**：
- AI 生成标题和描述
- 图片处理（抠图、生成）
- 价格策略（成本计算、竞品对标）

**MCP 工具**：
| 工具 | 用途 | 频率限制 |
|------|------|----------|
| `ozon_pricing` | 计算最优售价 | ≤10次/小时/商品 |
| `ozon_category` | 获取类目属性 | 无限制 |
| `ozon_listing` | 批量上架商品 | ≤80次/分钟 |
| `ozon_image_gen` | AI生成图片 | 视配额 |

### 3.3 MCP 协议集成

在 `electron-builder.yml` 或配置文件中添加：

```json
{
  "mcpServers": {
    "ozon-selection": {
      "command": "npx",
      "args": ["@haloclaw/ozon-selection-mcp"]
    },
    "source-1688": {
      "command": "npx",
      "args": ["@haloclaw/source-1688-mcp"]
    },
    "ozon-listing": {
      "command": "npx",
      "args": ["@haloclaw/ozon-listing-mcp"]
    }
  }
}
```

## 四、Supabase 用户认证

### 4.1 认证服务架构

```
src/main/services/auth.ts
├── HaloClawAuthService 类
│   ├── login()          # 用户登录
│   ├── register()       # 用户注册
│   ├── logout()         # 用户登出
│   ├── getCurrentUser() # 获取当前用户
│   └── syncToNewAPI()   # 同步到计费系统
└── registerAuthHandlers() # IPC 处理器
```

### 4.2 Supabase 配置

**连接信息**：
- URL: `https://kekmppsuiiokdckdeolv.supabase.co`
- Anon Key: 前端只读权限
- Service Key: 后端完整权限

**数据表结构**（参考 `n8n_supabase_setup.sql`）：

| 表名 | 用途 |
|------|------|
| `product_titles` | 产品标题和关键词 |
| `product_pricing` | 价格和利润数据 |
| `product_attributes` | 类目属性信息 |
| `ozon_categories` | Ozon 类目映射 |
| `user_quotas` | 用户配额（需扩展） |

### 4.3 IPC 接口

| 接口 | 参数 | 返回 |
|------|------|------|
| `auth:login` | `{email, password}` | `{success, data, error}` |
| `auth:register` | `{email, password, username?}` | `{success, data, error}` |
| `auth:logout` | - | `{success, error}` |
| `auth:getCurrentUser` | - | `User \| null` |
| `auth:checkStatus` | - | `{user, session, isAuthenticated}` |
| `auth:resetPassword` | `email` | `{success, error}` |
| `auth:getQuota` | - | `{quota, used}` |

## 五、NewAPI 计费集成

### 5.1 计费流程

```
用户注册/登录
    ↓
Supabase 认证成功
    ↓
自动同步用户到 NewAPI
    ↓
用户在 NewAPI 充值 Quota
    ↓
AI 调用时扣减 Quota
    ↓
显示剩余配额
```

### 5.2 Quota 配置

| 项目 | 默认值 |
|------|--------|
| 新用户配额 | 1000 Quota |
| 计费单位 | 1元 = 1000 Quota |
| 模型消耗 | Claude/GPT 按实际消耗计算 |

### 5.3 API 接口

**用户同步**：
```http
POST /users
Authorization: Bearer {ADMIN_KEY}
{
  "email": "user@example.com",
  "user_id": "uuid",
  "name": "username"
}
```

## 六、环境配置

### 6.1 .env 配置

```bash
# Supabase
SUPABASE_URL=https://kekmppsuiiokdckdeolv.supabase.co
SUPABASE_ANON_KEY=your_key
SUPABASE_SERVICE_KEY=your_key

# NewAPI
NEW_API_URL=https://your-newapi.com
NEW_API_ADMIN_KEY=your_admin_key

# Ozon
OZON_CLIENT_ID=your_client_id
OZON_API_KEY=your_api_key

# 1688
CHINESE_APP_KEY=your_key
CHINESE_APP_SECRET=your_secret
CHINESE_COOKIES=your_cookies
```

### 6.2 敏感信息管理

- **禁止硬编码**：所有密钥必须通过环境变量或 .env 文件
- **版本控制**：.env 文件必须加入 .gitignore
- **生产环境**：使用服务器环境变量或密钥管理服务
